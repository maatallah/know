# دليل الإعداد الفني: تثبيت منصة 'Know'

يشرح هذا الدليل كيفية نشر تطبيق "Know" (والذي يعمل كتطبيق Full-Stack Next.js يحتوي على كل من الواجهة الأمامية والخلفية) على خادم محلي، مثل **Windows Server** أو **QNAP NAS**.

## 1. المتطلبات الأساسية
- **Node.js**: الإصدار v18.17 أو أحدث (يُوصى بالإصدار v20).
- **PostgreSQL**: الإصدار v14 أو أحدث.
- **Git**: لسحب مستودعات النشر (اختياري ولكنه موصى به).

---

## 2. "ما يتعين علينا القيام به في الواجهة الأمامية"
نظراً لأن المنصة مبنية على Next.js، **يتم نشر الواجهة الأمامية (Frontend) والواجهة الخلفية (Backend) معاً كوحدة واحدة**. لا تحتاج إلى استضافة مستودع واجهة أمامية منفصل أو تكوين خادم ويب خاص بالواجهة الأمامية (مثل Apache/Nginx) فقط لخدمة الملفات الثابتة.

أثناء عملية التثبيت، ستقوم أوامر `npm run build` تلقائياً بما يلي:
1. تجميع كود React للواجهة الأمامية.
2. تحسين جميع الصور والخطوط وملفات CSS (بما في ذلك السمة العتيقة Vintage Theme).
3. تجميع مسارات واجهة برمجة التطبيقات (API) للواجهة الخلفية.

وبالتالي، فإن "نشر الواجهة الأمامية" هو ببساطة خطوة 'البناء' في هذا الإعداد الموحد.

---

## 3. خطوات التثبيت (عامة)

### الخطوة 1: إعداد قاعدة البيانات
1. قم بتثبيت PostgreSQL على الخادم الخاص بك (أو استخدم مركز تطبيقات QNAP لتثبيت PostgreSQL داخل حاوية).
2. قم بإنشاء قاعدة بيانات فارغة جديدة باسم `know_db`.
3. قم بإنشاء مستخدم قاعدة بيانات بكلمة مرور آمنة ومنحه الصلاحيات الكاملة على `know_db`.

### الخطوة 2: إعداد الكود وبيئة العمل
1. انسخ الكود المصدري إلى الخادم الخاص بك (مثلاً `C:\deployments\know-app` على أنظمة Windows).
2. قم بإنشاء ملف `.env` في المجلد الجذري مع المتغيرات التالية:

```env
# اتصال قاعدة البيانات (استبدل البيانات ببياناتك الفعلية)
DATABASE_URL="postgresql://USERNAME:PASSWORD@localhost:5432/know_db?schema=public"

# إعدادات NextAuth
NEXTAUTH_URL="http://IP_الخادم_الخاص_بك:3000"
NEXTAUTH_SECRET="قم-بإنشاء-نص-عشوائي-من-32-حرفا-هنا"
```

### الخطوة 3: البناء والترحيل
افتح موجه الأوامر (Command Prompt أو PowerShell كمسؤول) في مجلد التطبيق:

```bash
# 1. تثبيت التبعيات
npm install

# 2. دفع مخطط قاعدة البيانات
npx prisma db push

# 3. بذر البيانات الأولية (إنشاء أول مشرف عام Super Admin)
npm run seed

# 4. بناء الواجهة الأمامية والخلفية
npm run build
```

## 4. تكوينات الاستضافة

### الخيار أ: خادم Windows (يوصى باستخدام PM2)
لبقاء التطبيق يعمل باستمرار في الخلفية على أنظمة Windows:

1. قم بتثبيت PM2 عالميًا: `npm install -g pm2`
2. قم بتثبيت خدمة PM2 لنظام Windows: `npm install -g @innocenzi/pm2-windows-service`
3. قم بتشغيل التطبيق:
   ```bash
   pm2 start npm --name "know-app" -- start
   ```
4. احفظ قائمة PM2 لكي يتم إعادة التشغيل عند إعادة تشغيل النظام:
   ```bash
   pm2 save
   ```
*(اختياري: استخدم خادم IIS كوكيل عكسي (Reverse Proxy) لتوجيه حركة المرور عبر المنفذ 80/443 إلى localhost:3000).*

### الخيار ب: أجهزة التخزين الشبكي QNAP NAS
تعتبر أجهزة QNAP مضيفات ممتازة تتطلب صيانة قليلة. يمكنك النشر باستخدام تطبيق **Container Station** الخاص بـ QNAP (يعتمد على Docker).

1. تأكد من تثبيت تطبيق **Container Station** على جهاز QNAP.
2. في المجلد الجذري للمشروع، تأكد من وجود ملف `Dockerfile` قياسي لتطبيق Next.js.
3. اتصل عبر SSH بجهاز QNAP أو استخدم واجهة مستخدم Container Station لبناء وتشغيل صورة Docker:
   ```bash
   docker build -t know-app .
   docker run -d -p 3000:3000 --name know-service --env-file .env know-app
   ```

## 5. تسجيل الدخول الأول
بمجرد تشغيل التطبيق، انتقل إلى الرابط `http://IP_الخادم_الخاص_بك:3000`.
- قم بتسجيل الدخول باستخدام بيانات اعتماد المشرف الافتراضية (والمحددة في ملف `prisma/seed.ts`).
- انتقل فوراً إلى إعدادات المستخدم / الأدوار لإنشاء حسابات المديرين والفنيين الفعلية الخاصة بك.
